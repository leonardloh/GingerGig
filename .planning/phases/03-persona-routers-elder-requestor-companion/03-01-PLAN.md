<objective>
Create the Phase 3 persona-router verification harness that later plans must satisfy. Tests should exercise real FastAPI routes through `httpx.AsyncClient`, authenticate by logging in the seeded demo users through `/api/v1/auth/login`, and assert frontend-compatible camelCase response contracts, authorization failures, locale projection, seeded match fields, and mutation behavior.

TDD mode is false, so this plan does not require red-first commits. The executor may land these tests together with minimal skips/xfails only if necessary to keep the suite green during Wave 0, but each skip/xfail must be removed by the implementation plan that satisfies it.
</objective>

<threat_model>
## Security Threat Model

**Assets:** elder listings/bookings/earnings, requestor booking history, companion-watched elder data, bearer tokens, locale-specific alert/timeline content.

**Threats and mitigations:**
- **Cross-persona data exposure:** tests must prove elder, requestor, and companion-only endpoints reject role mismatches with `403`.
- **Cross-user access:** tests must prove users cannot access another elder's listings/bookings or an unlinked companion elder.
- **Invalid mutation state:** tests must prove booking response rejects non-pending bookings with `409`.
- **Error envelope drift:** tests must assert rejected persona requests return `{status, message, detail?}` rather than raw FastAPI traceback/default shapes.
- **Unauthorized access:** tests must assert missing bearer tokens return `401`.
</threat_model>
