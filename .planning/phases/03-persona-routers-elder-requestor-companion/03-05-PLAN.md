<objective>
Replace the companion router stub with protected DB-backed endpoints for watched elder dashboard, alerts, timeline, and alert preferences. All companion reads must validate `companion_links`, project locale-specific alert/timeline text at SQL level with English fallback, and preserve seeded demo content.
</objective>

<threat_model>
## Security Threat Model

**Assets:** watched elder profile snapshot, earnings/activity summary, companion alerts, timeline events, alert preference toggles.

**Threats and mitigations:**
- **Unlinked companion access:** every companion elder route must require a `companion_links` row for `(current_user.id, elderId)`.
- **Role mismatch:** elder and requestor users must receive `403` on companion routes.
- **Locale leakage:** alerts and timeline must select only one locale expression with English fallback at SQL construction time.
- **Preference tampering:** `PUT /alert-preferences` must only update the linked companion/elder row.
- **Frontend contract drift:** preferences route must return `204 No Content` with an empty body.
</threat_model>
