<objective>
Build the shared Phase 3 foundation: frontend-shaped persona DTOs, SQL-level locale helpers, mapper/query helpers, KL-month earnings utilities, and DB-backed seeded requestor match fields. This plan deliberately persists `distance`, `matchScore`, and locale-aware `matchReason` on listings so requestor search remains deterministic and database-backed without live Qwen calls.
</objective>

<threat_model>
## Security Threat Model

**Assets:** persisted listing metadata, seeded demo match explanations, persona response DTOs, booking snapshot data.

**Threats and mitigations:**
- **Runtime dependence on seed constants:** routers/services must read match data from Postgres, not import `backend/scripts/seed_data.py` or frontend mock data.
- **AI exfiltration/cost risk:** Phase 3 must not call Qwen, DashScope, OpenAI, Redis/Tair, KYC, voice, payment, or notification services.
- **Locale leakage:** helpers must select one locale expression with English fallback at SQL construction time; they must not fetch all locale columns for client-side selection.
- **Invalid match score data:** migration must constrain `match_score` to `0..100` when non-null.
- **Schema drift:** Alembic migration, SQLAlchemy model, seed upsert, and migration tests must agree on new columns.
</threat_model>

Create `backend/app/services/persona_queries.py` with the full shared helper surface required by all Wave 2 router plans. This task is the sole owner of `persona_queries.py`; later Wave 2 plans must consume these helpers and must not add new helper functions to this file.

Include reusable helpers:
- `locale_column(model, stem: str, locale: str)` using fixed mappings for `ms`, `en`, `zh`, and `ta`; reject unknown locale by falling back to English.
- `locale_expr(model, stem: str, locale: str, label: str | None = None)` returning `func.coalesce(locale_column(...), english_column).label(label or stem)`.
- `initials(name: str) -> str` returning uppercase initials from the first two words, with one-word fallback.
- `require_role(user: User, role: str) -> None` raising `HTTPException(status_code=403, detail="Forbidden")`.
- `require_self(user: User, user_id: UUID) -> None` raising `403` when `str(user.id) != str(user_id)`.
- `require_companion_link(db, companion_id, elder_id) -> None` querying `CompanionLink` and raising `403` if absent.
- `menu_items_for_listings(db, listing_ids)` batching menu items by listing ID.
- `reviews_for_listing(db, listing_id)` returning frontend-shaped review schema objects.
- `booking_to_response(booking)` mapping denormalized booking DB fields to `Booking`.
- listing response mappers used by elder, requestor search, and listing detail routes, including menu attachment, elder public snapshot fields, DB-first `distance`/`matchScore`/`matchReason`, and locale-projected titles.
- booking snapshot helpers for requestor booking creation, including deterministic `quantity_label`, `item_description`, `amount`, and locale-projected `listing_title` from a listing/menu row.
- `month_window_kl(now: datetime | None = None)` using `ZoneInfo("Asia/Kuala_Lumpur")` to return UTC start/end instants for the current KL calendar month.
- last-7-days KL/UTC window or equivalent helper needed by companion weekly dashboard aggregates.
- deterministic fallback helpers for requestor search: `fallback_match_score(rating, distance_label) -> int` and `fallback_match_reason(locale) -> str`, used only when DB columns are null.

The helpers may return dictionaries or `persona.py` schema objects, but they must keep route handlers thin and avoid N+1 menu/review queries. If a Wave 2 implementer discovers a missing shared helper, update this plan before execution rather than editing `persona_queries.py` from the Wave 2 router plans.
