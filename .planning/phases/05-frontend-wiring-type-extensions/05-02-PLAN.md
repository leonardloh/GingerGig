<objective>
Replace in-memory login/register behavior with the existing typed auth endpoints while preserving the visual quick-login chips, login form, app shell, onboarding stepper, and local KYC demo flow. Keep bearer token storage inside `auth.ts` / `http.ts`; `PrototypeApp.jsx` stores the returned `accessToken` only as session data to pass to `ElderVoice` later.
</objective>

<threat_model>
Frontend risks for this wave:
- **Token handling:** `login()` and `register()` must remain the only functions that call `setApiAccessToken`; signout must call `logout()` before clearing local state.
- **Wrong persona / IDOR-prone calls:** route persona from `getMe().role`, not from trusted client-side form/chip metadata, while keeping chip labels unchanged.
- **CORS/env:** auth calls must use `apiRequest` through `endpoints/auth`; no hardcoded backend origin or manual `/api/v1` prefix in JSX.
- **No UI drift:** no copy, class names, CSS, tab structure, layout, or KYC visual flow changes.
</threat_model>
