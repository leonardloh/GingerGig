<objective>
Remove `mock-data.js` imports from elder dashboard, listings, earnings/profile surfaces, replacing them with typed endpoint calls and screen-local adapters while preserving existing JSX structure, copy, styling, class names, and local animation behavior. `ElderVoice` remains for plan 05-06.
</objective>

<threat_model>
Frontend risks for this wave:
- **Wrong persona / IDOR-prone calls:** elder endpoints must use `user.id` from authenticated `getMe()` session props; do not use hardcoded elder ids for elder-owned routes.
- **Token handling:** do not read or mutate tokens here; endpoint modules use the existing bearer token installed by auth.
- **CORS/env:** use endpoint helpers only; no manual `fetch`, no hardcoded backend origin, no manual `/api/v1`.
- **No UI drift:** adapters must feed existing display props; do not edit CSS, class names, layout, copy, tab definitions, or the voice flow in this plan.
</threat_model>
