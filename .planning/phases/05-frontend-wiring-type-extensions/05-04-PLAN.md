<objective>
Remove `mock-data.js` imports from requestor home, search, detail, and profile screens. Drive listings from `searchListings()` and listing detail from `getListingById()`, using local adapters to preserve the existing provider-shaped JSX and visual behavior.
</objective>

<threat_model>
Frontend risks for this wave:
- **Wrong ID / IDOR-prone calls:** search cards must pass backend `listing.id` into detail, not old mock provider ids such as `siti` or `chen`.
- **Token handling:** do not inspect or mutate bearer tokens; endpoint helpers use the existing API client.
- **CORS/env:** do not add manual `fetch`, hardcoded API origins, or `/api/v1` in JSX.
- **No UI drift:** keep search/detail/profile layout, buttons, copy, badges, and requestor voice modal unchanged; only data source and adapter logic change.
</threat_model>
