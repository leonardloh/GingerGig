<objective>
Remove `mock-data.js` imports from companion screens, use companion-safe endpoints for dashboard, alerts, timeline, and alert preferences, and isolate the watched elder id bridge to the seeded Faiz -> Siti demo account if no API discovery route exists. Do not call elder-only routes with a companion token.
</objective>

<threat_model>
Frontend risks for this wave:
- **IDOR-prone companion calls:** companion screens must only call `/companions/elders/{elderId}/...`; never call `/elders/{elderId}/...` using a companion token.
- **Watched elder id discovery:** if no backend endpoint exposes watched elders, the bridge must be a single clearly named demo mapping for `faiz@gingergig.my -> entity_id("user", "siti")`, not a general client-side authorization mechanism.
- **Token handling:** do not inspect or persist tokens in companion screens; auth is handled by the existing API client.
- **No UI drift:** keep companion dashboard, alerts, profile copy, CSS, layout, tones, and inline decorative/live-feed content unchanged unless replacing imported mock constants with API-backed data.
</threat_model>
