<objective>
Extend the typed frontend API contract additively, add missing endpoint helpers for listing detail, companion timeline, and voice-to-profile, and export the new helper module. This wave creates stable names and DTO shapes before any prototype screen imports change. Preserve `apiRequest`, `setApiAccessToken`, `ApiError`, and `/api/v1` prefix behavior exactly.
</objective>

<threat_model>
Frontend risks for this wave:
- **Token handling:** WebSocket auth must receive the token explicitly and place it only in the Phase 4 required `?token=` query string; do not add persistent token storage or log tokens.
- **CORS/env/double prefix:** ordinary HTTP helpers must call `apiRequest("/...")` and must not concatenate `/api/v1`; only WebSocket and S3 direct `PUT` bypass `apiRequest`.
- **DTO drift:** `types.ts` must match backend `schemas/auth.py`, `schemas/persona.py`, and Phase 4 `schemas/voice.py` contracts without renaming or deleting existing fields.
- **No UI drift:** this plan touches API client files only; no prototype JSX, CSS, class names, copy, or layout.
</threat_model>
