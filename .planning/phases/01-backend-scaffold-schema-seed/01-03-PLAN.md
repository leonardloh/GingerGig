|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | T (Tampering) | Application code writing invalid enum values to constrained columns (e.g. `role='admin'`) | mitigate | Postgres CHECK constraints reject the row at INSERT time. Phase 3+ also use the StrEnum mirrors in `app.core.enums` for type safety at the Python layer. |
| T-03-02 | T | Booking row's listing-title or requestor-name being mutated by a later listing/user edit (loss of audit integrity) | mitigate | DATA-04 denormalisation: `bookings.{requestor_name, requestor_initials, requestor_avatar_url, listing_title, quantity_label, item_description}` are snapshots at create-time. | T-03-03 | I (Information Disclosure) | KYC raw Textract/Rekognition responses (which may contain partial IC numbers, face-bounding-box coords) being read by routers that don't need them | accept | `textract_raw` and `rekognition_raw` are JSONB on `kyc_sessions`. | T-03-04 | E (EoP) | A listing or alert with NULL where the schema demands NOT NULL slipping through into the API as `null` and crashing the typed frontend | mitigate | All columns marked `nullable=False` (NOT NULL) at the DB level. | T-03-05 | I | Cross-table slug collisions causing two seeded entities to share the same `id` (e.g. user "siti" and listing "siti") | mitigate | Plan 01's `entity_id(kind, slug)` uses `kind:slug` prefix in the UUID5 derivation — `entity_id("user","siti") != entity_id("listing","siti")`.
