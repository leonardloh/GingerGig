    ```python
    from contextlib import asynccontextmanager
    from typing import Any

    from fastapi import APIRouter, FastAPI
    from fastapi.middleware.cors import CORSMiddleware

    from app.core.config import settings
    from app.core.errors import register_exception_handlers
    from app.db.session import build_engine, dispose_engine
    from app.routers import (
        auth as auth_router,
        companion as companion_router,
        elder as elder_router,
        health as health_router,
        kyc as kyc_router,
        requestor as requestor_router,
        voice as voice_router,
    )


    @asynccontextmanager
    async def lifespan(app: FastAPI):
        app.state.engine = build_engine(settings)
        yield
        await dispose_engine(app.state.engine)


    app = FastAPI(
        title="GingerGig API",
        version="0.1.0",
        debug=False,  # Pitfall 20 — never True in any environment
        lifespan=lifespan,
    )

    # CORS — explicit allowlist, never "*" (Pitfall 19, FOUND-03)
    app.add_middleware(
        CORSMiddleware,
        allow_origins=settings.cors_origins,
        allow_credentials=False,  # bearer token, no cookies
        allow_methods=["*"],
        allow_headers=["*"],
    )

    register_exception_handlers(app)

    # /health is mounted WITHOUT /api/v1 prefix per RESEARCH (FOUND-01 success criterion)
    app.include_router(health_router.router)

    API = "/api/v1"
    app.include_router(auth_router.router, prefix=API)
    app.include_router(elder_router.router, prefix=API)
    app.include_router(requestor_router.router, prefix=API)
    app.include_router(companion_router.router, prefix=API)
    app.include_router(kyc_router.router, prefix=API)
    app.include_router(voice_router.router, prefix=API)

    # Test-only route exercising the global Exception handler (D-17 #4).
    # Mounted under /__test__ so production-mode tooling can identify and exclude it.
    _test_router = APIRouter(prefix="/__test__", tags=["__test__"])


    @_test_router.get("/boom")
    async def _boom() -> dict[str, Any]:
        raise RuntimeError("kaboom")  # noqa: TRY003 — intentional test fixture


    app.include_router(_test_router)
    ```

    Note on `__test__` route: it is intentionally mounted in main.py (not behind an env-var guard) so Plan 06's `tests/test_error_envelope.py` can exercise the unhandled-exception path. The route is a single line that raises; PII risk is zero. If the developer wants to exclude it post-Phase-1, that becomes a Phase 8 deploy concern; it's tagged `__test__` so a future filter is trivial.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser (CloudFront / Vite dev) → FastAPI | CORS preflight + Bearer header (Phase 2 verifies); Phase 1 only protects against wildcard CORS and traceback leaks |
| FastAPI process → ApsaraDB | TLS enforced by Plan 01's engine builder |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | I | CORS wildcard `*` enabling cross-origin token theft post-Phase-2 | mitigate | `CORSMiddleware(allow_origins=settings.cors_origins, allow_credentials=False)` — `cors_origins` is the parsed list from `cors_origins_csv`; never includes `"*"`. Plan 06 has `test_cors_no_wildcard.py` enforcing this. |
| T-02-02 | I | Python traceback leaked in 5xx response body | mitigate | `app.core.errors.unhandled_exception_handler` returns hard-coded `{"status":500, "message":"Internal server error"}`. The `__test__/boom` route exercises this path; Plan 06 verifies `"Traceback" not in response.text`. |
| T-02-03 | I | Validation error detail leaking sensitive request data (e.g. password values) | mitigate | `validation_exception_handler` returns `exc.errors()` which is the structured Pydantic error list (field names + types + messages, not raw values). For Phase 2 password endpoints, the `RegisterPayload`/`LoginPayload` field validation messages are safe. |
| T-02-04 | T | Stub route accepting arbitrary input that reaches a Phase-2-incomplete handler | accept | Stubs return 501 immediately via `HTTPException`; no body parsing. Worst case is information disclosure of the route existing — required for Phase 2-5 to mount on top. |
| T-02-05 | E | `__test__/boom` route enabling DoS by error-log spam | accept | Single route, single line, hackathon scope. Production CORS allowlist limits exposure to whitelisted origins. Phase 8 may filter `__test__` routes from the deployed app via lifespan check; not Phase 1's concern. |
| T-02-06 | I | Lifespan engine build failure leaking the connection string in stderr | mitigate | `Settings` field validation strips secrets from stack traces; asyncpg `InvalidCatalogNameError` doesn't include the password. Acceptable for dev; Phase 8 hardening optional. |

All `mitigate` threats meet ASVS L1 for CORS, error-handling, and validation messaging.
</threat_model>
