Harden the live cross-cloud deployment before the final smoke test: backend CORS allowlist, S3 audio CORS, WebSocket timeout/upgrade behavior, IAM scope, DashScope credential readiness, and no committed secrets. Block hardening against live origins until both `.planning/phases/04-voice-to-profile-pipeline/04-05-SUMMARY.md` and `.planning/phases/05-frontend-wiring-type-extensions/05-07-SUMMARY.md` exist.
</objective>

<threat_model>
## Security Threat Model

**Assets:** browser-to-backend API traffic, WebSocket audio path, S3 audio bucket, cloud credentials.

**Threats and mitigations:**
- **CORS wildcard:** backend and S3 CORS must use exact origins, never `*`.
- **WebSocket disconnect during demo:** load balancer/reverse proxy idle timeout must be >=300 seconds and upgrade headers must pass.
- **Credential leak:** repo scan must confirm real secrets are not committed.
- **IAM overreach:** AWS permissions remain scoped to the audio bucket and Transcribe actions needed by Phase 4.
</threat_model>
