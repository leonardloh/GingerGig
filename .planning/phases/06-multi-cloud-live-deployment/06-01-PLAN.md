<objective>
Provision and document the AWS side that is safe to prepare before Phase 5 completes: private S3 frontend bucket behind CloudFront OAC in `ap-southeast-1`, S3 audio bucket for batch ASR with browser PUT CORS, least-privilege IAM for Transcribe/S3, and AWS Budgets alerts at $50 and $100. Do not publish final frontend artifacts yet; that happens in 06-04 after Phase 5 is complete.
</objective>

<threat_model>
## Security Threat Model

**Assets:** frontend static app, audio objects, AWS IAM credentials, AWS account budget.

**Threats and mitigations:**
- **Public S3 bucket exposure:** frontend bucket must have Block Public Access enabled and be reachable only through CloudFront OAC.
- **Overbroad audio bucket CORS:** audio bucket CORS must list exact CloudFront/backend origins, not `*`.
- **Overbroad IAM:** backend AWS principal must be scoped to the named audio bucket/prefix and required Transcribe operations only.
- **Cost runaway:** AWS Budgets actual-cost alerts at $50 and $100 must be configured before judge traffic or repeated Transcribe tests.
</threat_model>
