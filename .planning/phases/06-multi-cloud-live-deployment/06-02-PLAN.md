<threat_model>
## Security Threat Model

**Assets:** backend service, database credentials, provider photos, DashScope API key, public backend origin.

**Threats and mitigations:**
- **Database exposure:** ApsaraDB must allow inbound only from the ECS/security-group path, not the public internet.
- **WebSocket demo failure:** SLB/ALB/reverse proxy timeout must be verified at >=300 seconds for voice sessions.
- **Secret leakage:** runbook uses placeholders only and never commits real `DATABASE_URL`, `JWT_SECRET`, `DASHSCOPE_API_KEY`, or OSS keys.
- **Media boundary drift:** OSS stores non-PII provider photos only; audio remains AWS S3 and KYC raw images remain AWS/Phase 4 scope.
</threat_model>
