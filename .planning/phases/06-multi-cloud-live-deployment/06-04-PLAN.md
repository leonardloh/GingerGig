<objective>
Publish the Phase 5-wired frontend to AWS S3 + CloudFront with `VITE_API_BASE_URL` pointing to the deployed Alibaba backend origin. Keep the API base URL as an origin only, with no trailing slash and no `/api/v1`. Block publication until both `.planning/phases/04-voice-to-profile-pipeline/04-05-SUMMARY.md` and `.planning/phases/05-frontend-wiring-type-extensions/05-07-SUMMARY.md` exist.
</objective>

<threat_model>
## Security Threat Model

**Assets:** deployed frontend bundle, backend origin, bearer token traffic.

**Threats and mitigations:**
- **Wrong API origin:** production `VITE_API_BASE_URL` must not include `/api/v1`; `apiRequest` appends it.
- **Premature publish:** block until Phase 5 final summary exists and backend public health passes.
- **Stale static assets:** CloudFront invalidation must run after S3 upload.
- **Accidental secret exposure:** Vite env may include only public `VITE_` variables and no backend secrets.
</threat_model>
