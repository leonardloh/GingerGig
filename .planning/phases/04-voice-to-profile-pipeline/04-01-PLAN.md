<threat_model>
## Security Threat Model

**Assets:** elder-owned `voice_sessions` rows, S3 object keys, Transcribe job names, JWTs in WebSocket query strings.

**Threats and mitigations:**
- **IDOR on batch job polling:** `jobId` must map to a `voice_sessions` row where `elder_id == current_user.id`; contract tests will assert the route exists and 04-04 implements ownership checks.
- **boto3 misuse for streaming:** static guardrails ensure streaming integration never imports `boto3` for Transcribe streaming APIs.
- **Schema drift vs. runtime:** migration must run before any test that inserts batch correlation data.
- **Secret leakage in errors:** contract tests will later assert 502 user-safe messages; this plan only anchors paths and import rules.
</threat_model>
