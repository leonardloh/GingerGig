<objective>
Introduce `app/schemas/voice.py` with API DTOs for voice and a `ListingDraft` model that preserves the Qwen/architecture snake_case fields (`service_offer`, `price_amount`, etc.), implement `app/services/qwen_service.py::extract_listing` using the OpenAI-compatible client with `response_format={"type":"json_object"}`, markdown fence stripping, Pydantic `ListingDraft` validation, exactly one retry on `ValidationError` with the error text appended, and 502 with message `Listing extraction failed` on persistent failure. Add focused `tests/test_voice_qwen.py` with mocks only (no live DashScope). Ensure `DASHSCOPE_API_KEY` / `dashscope_base_url` remain documented in `.env.example`.
</objective>

<threat_model>
## Security Threat Model

**Assets:** Qwen API key, raw model JSON in memory.

**Threats and mitigations:**
- **Key exfil via logs:** do not log full API responses or transcripts in production; tests use fake keys via env in test settings.
- **Prompt injection from transcript:** treat transcript as untrusted; extraction output is Pydantic-bound only, no `eval` of arbitrary code.
- **502 body leakage:** return only `message: Listing extraction failed` in HTTP layer (envelope from `core/errors.py`); `extract_listing` may raise a dedicated exception or return `Result` type consumed by routers in 04-03/04-04.
</threat_model>
