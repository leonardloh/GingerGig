<objective>
Implement real-time voice-to-listing for en-US and zh-CN using `amazon-transcribe` (package `>=0.6.4`, not boto3) in `ap-southeast-1`, a WebSocket at `GET /api/v1/voice-to-profile/stream?token=<JWT>`, 16kHz little-endian mono PCM in binary frames after an initial JSON handshake, partial JSON messages throttled to at most 4 per second (250ms minimum spacing) with duplicate consecutive partial text removed, 90s max session duration, and try/finally that always ends the upstream stream (`input_stream.end_stream()`) and cancels the partial reader task. Finalize with `qwen_service.extract_listing` and send `{"type":"final","listing":<ListingDraft>}` as text JSON. Create `app/integrations/transcribe_streaming.py` and `app/services/voice_service.py` orchestration, replace `voice.py` stub, add `tests/test_voice_streaming.py` with all AWS mocked.
</objective>

<threat_model>
## Security Threat Model

**Assets:** elder identity, transcription text, listing draft in WebSocket frames, AWS Transcribe session.

**Threats and mitigations:**
- **Token in query string:** only over WSS in production; reject missing or invalid `token` with WebSocket close code 4401 or prior HTTP 403 per FastAPI behavior; same `get_current_user_ws` as `app/deps/auth.py`.
- **Non-elder abuse:** call `require_role(user, "elder")` after auth.
- **Wrong language on stream:** reject `ms-MY` / `ta-IN` on the WebSocket with close code in 4xxx and reason `batch only` (ascii).
- **Resource exhaustion:** 90s watchdog; throttle partials; finally block always runs.
</threat_model>
